# Платежи

## Сценарии

Бот может обрабатывать покупку, продление, Telegram Stars, оплату через внешний провайдер, ручное подтверждение админом и retry для apply_failed.

## Статусы платежа

- `pending` — ожидает оплату;
- `processing` — оплата получена, подписка применяется;
- `success` — применено успешно;
- `failed` — ошибка или отмена.

## application_status

Отдельное состояние применения подписки: `not_applied`, `applied`, `apply_failed`, `reprocessing`. Это нужно, чтобы не создавать дубли подписок при повторной обработке.

## result_payload

После успешного применения может сохраняться результат: client name, subscription link, Clash link, tariff, period и другие данные. Это помогает повторно показать результат.

## Telegram Stars

Telegram Stars используется как отдельный способ оплаты. Цены Stars настраиваются в тарифе и периодах.

## Внешний платежный провайдер

Внешний провайдер может использоваться для карты/СБП. Для безопасности payload лучше делать коротким и не раскрывать бизнес-семантику.

## Что нельзя менять без проверки

В production не стоит без необходимости менять payment apply logic, result_payload, retry/apply_failed, provider status mapping и callback security.
