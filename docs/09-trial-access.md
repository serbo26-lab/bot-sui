# Тестовый доступ

Тестовый доступ — это отдельный режим выдачи временной подписки без оплаты.

Он нужен, чтобы пользователь мог проверить работу сервиса до покупки.

## Где находится

Пользователь:

```text
🎁 Тестовый доступ
```

Админ:

```text
🛠 Админ → 🎁 Тестовый доступ
```

## Что видит пользователь

Если кнопка включена и пользователь имеет право получить тест:

1. Пользователь нажимает `🎁 Тестовый доступ`.
2. Бот создает тестовую подписку.
3. Бот отправляет тариф, срок, ссылку подписки и кнопку `📱 Подключить устройство`.
4. Пользователь добавляет subscription link в VPN-клиент.

Если у пользователя уже есть активный тест или подписка, кнопка может быть скрыта или бот может отказать в выдаче.

## Показ кнопки

Админ может включить или выключить кнопку теста.

Если кнопка выключена, пользователь не видит `🎁 Тестовый доступ` в меню.

## Срок теста

Админ задает срок, например:

```text
1 день
3 дня
7 дней
10 дней
```

После окончания срока тест считается неактивным.

## Трафик теста

Можно задать число ГБ или `∞`.

`∞` означает безлимит. Для S-UI/sing-box это обычно соответствует `volume = 0`.

## Скрытие трафика

Если тест remote-only и traffic limit не имеет смысла, строку трафика можно скрыть из пользовательского сообщения.

## Лимит устройств/IP

Trial имеет свой лимит устройств/IP. Этот лимит используется в пользовательской карточке, antiabuse и предупреждениях.

## MAIN / MultiHop для теста

Админ может включить или выключить main/S-UI inbounds для теста.

Если main выключен, тест может быть remote-only.

## Test inbounds

Для теста можно задать отдельные S-UI inbounds.

Это полезно, если тест должен использовать не те же inbounds, что платные тарифы.

## Применение test inbounds к активным тестам

Если админ изменил test inbounds, бот может применить изменения к уже активным тестовым ключам через S-UI editbulk.

## Remote locations для теста

Тест может использовать remote nodes.

Настройки:

```text
• все node;
• выбранные node;
• без remote node;
• remote-only.
```

Это позволяет дать тест только на одну локацию, даже если платные тарифы имеют больше локаций.

## Повторная выдача

Админ может разрешить или запретить повторную выдачу теста.

Варианты:

```text
один раз навсегда
повторно после удаления/истечения
повторно после cooldown
```

## Cooldown

Cooldown ограничивает частоту повторной выдачи.

Например:

```text
не чаще одного раза в 30 дней
```

## Описание теста

Админ может задать пользовательское описание.

Лучше писать кратко:

```text
Тестовый доступ на 3 дня. Подходит для проверки скорости и подключения.
```

## Antiabuse для теста

Trial участвует в antiabuse как платные подписки.

Если активных логических IP больше лимита, бот может отправить warning админу и пользователю.

## Telegram proxy для теста

В `💎 Telegram proxy` админ может выбрать:

```text
только обычные подписки
обычные подписки + активный тестовый доступ
```

Если включен второй режим, trial-пользователь увидит proxy.

## Рассылка тестам

В `📣 Рассылка` можно выбрать аудиторию:

```text
активный тестовый доступ
обычные подписки + активный тестовый доступ
```

## Что проверить после настройки теста

```text
1. Кнопка теста скрывается/показывается.
2. Тест создается.
3. Срок правильный.
4. Трафик правильный.
5. ∞ отображается как ∞.
6. Remote-only тест не показывает лишний main-текст.
7. Node scope работает.
8. Лимит устройств/IP работает.
9. Antiabuse видит тест как тест.
10. Telegram proxy доступен тесту, если разрешено.
```
