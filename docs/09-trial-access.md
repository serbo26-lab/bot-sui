# Тестовый доступ

Тестовый доступ позволяет пользователю проверить сервис до покупки.

## Настройки

Администратор может настроить кнопку теста, срок, трафик включая `∞`, лимит устройств/IP, повторную выдачу, cooldown, доступ к MAIN/Double VPN, remote nodes, node scope и описание.

## Безлимитный трафик

Если указано `∞`, в runtime это должно обрабатываться как безлимит. Для S-UI/sing-box обычно это означает `volume = 0`.

## Remote-only trial

Тест может быть выдан только на remote nodes без MAIN/S-UI inbounds. В этом случае пользователь получает remote links.

## Antiabuse

Тестовые ключи участвуют в antiabuse как обычные подписки. Лимит устройств/IP берется из настроек тестового доступа.

## Telegram proxy для теста

Telegram proxy может быть доступен только обычным подпискам или обычным подпискам + активному тестовому доступу.

## Автоудаление истекших тестов

Бот может удалять истекшие тестовые ключи после grace period, чтобы уменьшить мусор в S-UI.
