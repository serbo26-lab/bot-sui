# Глоссарий

## S-UI
Панель управления sing-box. В проекте используется как основной источник клиентов, подписок и API-интеграции.

## sing-box
Прокси/VPN-core, который обрабатывает подключения пользователей. Используется на основном сервере и remote node-серверах.

## Main server
Основной сервер проекта. На нем обычно находятся Telegram-бот, S-UI панель, SQLite-база, конфигурация, сертификаты-источник и systemd service.

## Remote node
Дополнительный VPS-сервер с sing-box, который используется как отдельная VPN-локация. Бот управляет remote node через SSH.

## Subscription link
Ссылка подписки, которую пользователь добавляет в VPN-клиент. Внутри могут быть main-сервер и remote node-локации.

## Double VPN
Отдельный управляемый режим доступа через основной сервер/S-UI inbounds. В разных установках смысл может отличаться.

## Inbounds
S-UI/sing-box inbound ID, которые используются для подключения клиента к main-серверу.

## External links
Ссылки remote node, которые бот добавляет к подписке S-UI-клиента.

## TGID
Telegram user ID. Надежнее username, потому что username может отсутствовать или измениться.

## Trial access
Тестовый доступ со своим сроком, лимитом трафика, лимитом устройств/IP и списком доступных node.

## Manual/offline subscription
Подписка, созданная администратором для клиента, который может не использовать Telegram.

## Antiabuse case
Карточка подозрения на превышение лимита устройств/IP.

## Dynamic IP dedupe
Механизм, который группирует похожие IPv4 адреса одного провайдера в один логический источник, чтобы уменьшить ложные предупреждения из-за динамической смены IP.

## legacy_domains
Старые домены node, которые используются для очистки старых managed links после миграции домена.
