# Глоссарий

## Bot S-UI

Telegram-бот для управления VPN-подписками через S-UI API и sing-box.

## S-UI

Панель управления sing-box. В проекте используется как основной источник клиентов, подписок и API-интеграции.

## sing-box

Прокси/VPN-core, который обрабатывает подключения пользователей.

## Main server

Основной сервер проекта: Telegram-бот, S-UI панель, SQLite-база, конфигурация и сертификаты-источник.

## Remote node

Дополнительный VPS-сервер с sing-box, который используется как отдельная VPN-локация.

## MultiHop / Double VPN

Внутреннее название режима, в котором трафик пользователя проходит цепочкой через несколько промежуточных точек. Название “Double VPN” появилось как рабочее и может не отражать все варианты маршрутизации.

Возможные схемы:

```text
пользователь → основной сервер → второй сервер → интернет
пользователь → основной сервер → второй сервер → WARP → интернет
пользователь → основной сервер → WARP endpoint → интернет
```

Этот режим может вообще не использоваться, если проекту достаточно обычных remote node-подписок.

## TGID

Telegram user ID. Надежнее username, потому что username может отсутствовать или измениться.

## Manual/offline subscription

Подписка, созданная администратором для клиента, который может не использовать Telegram.

## Dynamic IP dedupe

Группировка похожих IPv4 адресов одного провайдера в один логический источник, чтобы уменьшить ложные предупреждения из-за динамической смены IP.
