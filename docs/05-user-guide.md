# Пользовательские функции

Этот раздел описывает, что делает каждая пользовательская кнопка Bot S-UI. Названия могут отличаться, если администратор изменил тексты, но логика остается такой же.

## /start и /menu

Команды открывают главное меню. Приветствие может редактироваться администратором. В тексте приветствия можно использовать `{locations}`, чтобы показывать актуальные локации.

## 🛒 Купить

Открывает покупку подписки.

Сценарий:

1. Пользователь нажимает `🛒 Купить`.
2. Бот показывает доступные тарифы.
3. Пользователь выбирает тариф.
4. Бот показывает предпросмотр: срок, цену, способы оплаты, лимит устройств/IP, локации и MultiHop / Double VPN, если он включен.
5. Пользователь выбирает способ оплаты.
6. После оплаты бот создает подписку в S-UI, обновляет remote links и показывает subscription link.

Если у тарифа несколько сроков, кнопки оплаты сразу содержат срок:

```text
💳 · 30 дн. · Картой / СБП
💳 · 365 дней · Картой / СБП
⭐ · 30 дн. · Telegram Stars
⭐ · 365 дней · Telegram Stars
```

## 🔄 Продлить

Открывает продление существующей подписки.

Сценарий:

1. Пользователь выбирает подписку.
2. Бот показывает доступные сроки продления.
3. Пользователь оплачивает выбранный срок.
4. Бот продлевает существующего клиента, не создавая дубль.
5. Ссылка подписки остается прежней.

Продление использует условия тарифа: периоды, цены, скидки, remote node scope и MultiHop / Double VPN.

## 🔑 Мои подписки

Показывает активные подписки пользователя.

В карточке могут быть:

- тариф;
- срок действия;
- лимит устройств/IP;
- текущие устройства/IP;
- subscription link;
- Clash link;
- локации;
- MultiHop / Double VPN трафик, если режим включен.

Если MultiHop / Double VPN недоступен, строку лучше не показывать.

## 📱 Подключить устройство

Открывает инструкции подключения. Обычно содержит iPhone/iPad, Android, Windows, macOS, роутеры и другие платформы.

Эта кнопка не создает подписку, а объясняет, как использовать уже выданную ссылку.

## 💳 Мои платежи

Показывает историю платежей пользователя.

В карточке платежа могут быть:

- статус;
- сумма;
- способ оплаты;
- дата;
- покупка или продление;
- кнопки проверки/повтора для безопасных сценариев.

## 🎁 Тестовый доступ

Выдает trial-подписку, если функция включена администратором и пользователь имеет право получить тест.

Тест может иметь свои настройки: срок, трафик, `∞`, лимит устройств/IP, remote node scope, main/MultiHop и cooldown.

## 💎 Telegram proxy

Показывает Telegram proxy, если доступ разрешен.

Режим доступа задает админ:

```text
только обычные подписки
обычные подписки + активный тестовый доступ
```

Telegram proxy не заменяет VPN-подписку.

## 👤 Профиль

Показывает TGID, username и реферальную информацию.

TGID нужен, если админ хочет привязать manual/offline подписку к Telegram.

## 🤝 Реферальная программа

Обычно доступна через профиль. Пользователь получает ссылку, приглашает людей и копит скидку. Скидка применяется при покупке или продлении.

## 🆘 Поддержка

Открывает создание обращения. Пользователь отправляет сообщение, админ получает тикет и отвечает из админки.

## ❓ FAQ

Открывает пользовательскую справку. FAQ редактируется администратором из бота.
