# Antiabuse

Antiabuse отслеживает превышение лимита устройств/IP по подписке. Бот считает не физические устройства, а активные уникальные IP или логические источники IP.

## MAIN antiabuse

MAIN-часть читает логи S-UI/sing-box через `journalctl`, сопоставляет IP подключения, имя клиента, протокол и время события.

## Remote node antiabuse

На node может работать collector, который собирает summary по активным IP. Main-сервер периодически забирает summaries и объединяет данные с MAIN.

## Активное окно

Подозрение строится по активному окну, например 10 минут. Если активных логических источников больше лимита тарифа/теста, создается кейс и предупреждение.

## Лимит устройств/IP

Источник лимита: override в кейсе, trial settings, tariff connection limit или fallback из config.

## Dynamic IP dedupe

Некоторые провайдеры часто меняют полный IP, сохраняя первые октеты. Чтобы не создавать ложные предупреждения, бот может группировать похожие IPv4 в один логический источник:

```text
85.174.181.36
85.174.183.98
→ 85.174.*.*
```

Полные IP должны сохраняться в evidence для диагностики.

## Предупреждения

Предупреждения могут отправляться администратору и пользователю. Повтор предупреждения обычно происходит через заданный интервал, например 30 минут, пока превышение активно.

## Ручное отключение

Админ может вручную отключить подписку на 10 минут или на свой срок, а также включить ее обратно.

## Auto-disable

Auto-disable должен быть выключен по умолчанию. Безопасная логика: несколько подтверждений превышения, forced poll remote summaries, stale node data не используется, одинаковый IP на разных node считается один раз, после auto-enable действует cooldown, evidence сохраняется отдельно.

## Пользовательский текст

Пользователю не нужны технические детали. Достаточно сообщить, что подписка временно приостановлена из-за превышения лимита устройств/IP, указать срок и предложить обратиться в поддержку.
