# Manual/offline подписки

Manual/offline подписка — это полноценная подписка, которую создает администратор вручную.

Она нужна, если клиент не использует Telegram, оплатил вне автоматической системы, получил подписку от админа или сначала был offline-клиентом, а позже начал использовать Telegram.

## Чем отличается от “выдать без оплаты”

“Выдать без оплаты” обычно относится к Telegram-пользователю.

Manual/offline подписка может вообще не иметь TGID, username, Telegram-чата и пользовательских уведомлений.

При этом она остается полноценной подпиской и управляется ботом.

## Где находится

```text
🛠 Админ → 👤 Manual/offline подписки
```

## Что наследуется от тарифа

Manual/offline подписка привязывается к выбранному тарифу и наследует:

- срок;
- цены продления;
- лимит устройств/IP;
- remote node scope;
- доступные протоколы node;
- S-UI inbounds;
- MultiHop / Double VPN;
- antiabuse;
- renewal logic.

То есть это не “ручная ссылка вне системы”, а нормальная bot-managed подписка.

## Создание подписки

Типовой сценарий:

1. Админ открывает `👤 Manual/offline подписки`.
2. Нажимает `➕ Создать`.
3. Выбирает тариф.
4. Выбирает срок.
5. Вводит данные клиента.

Формат может быть таким:

```text
Имя/метка | контакт | заметка | Telegram
```

Пример без Telegram:

```text
Ivan Client | +79990000000 | оплатил наличными |
```

Пример с TGID:

```text
Ivan Client | +79990000000 | offline customer | 123456789
```

Пример с pending username:

```text
Ivan Client | @ivan_contact | привязать позже | @ivanvpn
```

## Имя / контакт / заметка

Эти поля нужны администратору для поиска и понимания, кому принадлежит подписка.

## Список подписок

Список должен иметь страницы, потому что manual/offline клиентов может быть много.

В списке полезно показывать ID, имя клиента, тариф, Telegram status и статус подписки.

## Поиск

Поиск должен работать по имени, контакту, заметке, client name, MANUAL_ID, TGID и username.

## Карточка manual/offline подписки

Карточка должна показывать:

- внутренний ID;
- `MANUAL_ID`;
- имя/контакт/заметку;
- Telegram binding;
- тариф;
- срок;
- дату окончания;
- статус;
- лимит устройств/IP;
- MultiHop / Double VPN;
- S-UI desc;
- кто создал;
- links.

## MANUAL_ID

В S-UI desc добавляется:

```text
MANUAL_ID=1
```

Даже если имя клиента изменится, MANUAL_ID сохраняет связь панели и карточки бота.

## Показать ссылки

Кнопка показывает subscription link и Clash link, если он поддерживается.

Это нужно, если клиент потерял ссылку.

## Изменить данные

Позволяет изменить имя/метку, контакт и заметку.

Эта операция не должна менять тариф, срок, node links, S-UI client ID и MANUAL_ID.

## Telegram binding

Кнопка `🔗 Telegram` позволяет привязать manual/offline подписку к Telegram.

Варианты ввода:

```text
123456789
123456789 @username
@username
-
```

`-` очищает привязку.

## Привязка по TGID

Самый надежный вариант. Пользователь может узнать TGID в разделе `👤 Профиль` и отправить его админу.

## Pending-привязка по username

Если админ знает только `@username`, бот сохраняет pending binding.

Когда пользователь запускает бота или открывает профиль, бот видит его настоящий TGID и username. Если username совпадает, подписка привязывается к TGID.

## После Telegram binding

После привязки пользователь получает преимущества обычной подписки:

- видит ее в `🔑 Мои подписки`;
- может продлевать по условиям тарифа;
- может получить Telegram proxy, если доступ разрешен;
- может получать user warning antiabuse;
- может получать напоминания.

## MultiHop / Double VPN override

Manual/offline подписка может использовать настройку тарифа, принудительно включить или принудительно выключить MultiHop / Double VPN.

Это полезно, если клиенту нужен отдельный режим, но общий тариф менять нельзя.

## Продлить

Продление manual/offline подписки использует выбранный тариф.

Если подписка привязана к Telegram, пользователь может продлить ее сам. Если не привязана, продлевает админ.

## Отключить / включить

Админ может отключить подписку.

MAIN/S-UI часть применяется сразу. Remote node-доступ обновляется после sync/deploy.

## Удалить

Удаляет manual/offline подписку из S-UI и managed remote data.

Перед удалением нужен confirm-экран.

## Antiabuse

Manual/offline подписка участвует в antiabuse.

Если TGID нет, предупреждения получает только админ. Если TGID есть, предупреждение может уйти пользователю.

## Что проверить

```text
1. Создание без TGID.
2. Создание с TGID.
3. Pending @username.
4. Автопривязка при входе пользователя.
5. Появление в “Мои подписки”.
6. Продление пользователем.
7. Telegram proxy после привязки.
8. Повторная выдача links.
9. Отключить/включить.
10. Удалить.
```
