# Тарифы и периоды

Тариф — это набор правил, по которым Bot S-UI создает и продлевает подписку.

Тариф отвечает не только за цену. Он определяет срок, лимит устройств/IP, доступные локации, S-UI inbounds, MultiHop / Double VPN и поведение в покупке/продлении.

## Где находится

```text
🛠 Админ → 💼 Тарифы
```

## Что можно сделать в разделе тарифов

Обычно раздел позволяет:

- посмотреть список тарифов;
- создать тариф;
- включить или выключить тариф;
- скрыть тариф из покупки;
- скрыть тариф из продления;
- изменить название;
- изменить название кнопки покупки;
- изменить описание;
- настроить цены;
- настроить сроки;
- настроить лимит устройств/IP;
- включить или выключить MultiHop / Double VPN;
- настроить S-UI inbounds;
- выбрать доступные remote nodes;
- настроить Telegram Stars.

## Создание тарифа

Типовой сценарий:

1. Открыть `🛠 Админ → 💼 Тарифы`.
2. Нажать `➕ Создать тариф`.
3. Указать технический код или название.
4. Открыть карточку тарифа.
5. Настроить пользовательское название.
6. Настроить сроки.
7. Настроить цены.
8. Настроить лимит устройств/IP.
9. Настроить remote nodes.
10. Проверить предпросмотр покупки.

После создания тариф может быть выключен или скрыт, пока админ не закончит настройку.

## Включение и выключение тарифа

Если тариф выключен, он не должен быть доступен пользователю для покупки.

Это полезно, если тариф еще настраивается или временно недоступен.

## Скрыть из покупки

Тариф может быть активен технически, но скрыт со страницы покупки.

Это удобно для старых тарифов, специальных тарифов, manual/offline клиентов или тарифов, которые нельзя купить публично.

## Скрыть из продления

Если тариф скрыт из продления, пользователь не сможет продлить подписку по этому тарифу через обычное меню.

Это удобно, если тариф больше не продается или должен продлеваться только админом.

## Название для пользователя

Используется в:

- предпросмотре покупки;
- оплате;
- продлении;
- `🔑 Мои подписки`;
- пользовательских сообщениях.

Пример:

```text
Максимальный
Семейный
Базовый
```

## Название для страницы тарифов

Отдельное название только для кнопки на странице покупки.

Пример:

```text
Кнопка покупки: 🚀 Максимальный · YouTube без рекламы
Название тарифа: Максимальный
```

Так можно сделать кнопку красивее, не загрязняя остальные экраны.

## Описание тарифа

Описание показывается в предпросмотре покупки.

Пример:

```text
Подходит для телефона, ноутбука и Smart TV.
```

Не стоит перегружать описание. Длинные инструкции лучше хранить в FAQ.

## Сроки тарифа

Тариф может иметь один или несколько сроков.

Пример:

```text
30 дней
365 дней
```

Если цена для срока не задана, этот срок не показывается пользователю.

## Как добавить срок 30 дней

1. Открыть карточку тарифа.
2. Открыть `🗓 Сроки`.
3. Добавить срок `30`.
4. Указать цену RUB.
5. Указать цену Stars, если нужно.
6. Сохранить.
7. Проверить предпросмотр покупки.

## Как добавить срок 365 дней

1. Открыть `🗓 Сроки`.
2. Добавить срок `365`.
3. Указать цену RUB.
4. Указать цену Stars, если нужно.
5. Проверить предпросмотр покупки.

Если цена 365 дней не задана, этот срок не показывается.

## Стоимость

В пользовательском предпросмотре цены лучше группировать по способу оплаты:

```text
Стоимость:

💳 Картой / СБП:
• 30 дн. — 100 ₽
• 365 дней — 1000 ₽

⭐ Telegram Stars:
• 30 дн. — 100 ⭐
• 365 дней — 1000 ⭐
```

## Telegram Stars

Для Stars могут быть отдельные настройки:

- включить/выключить Stars;
- цена Stars по каждому сроку;
- показывать Official/Alt варианты, если это реализовано.

Если Stars выключены или цена 0, кнопка Stars не показывается.

## RUB / карта / СБП

Оплата картой/СБП рассчитана на подключенный платежный провайдер, например Platega.

Если цена RUB не задана, кнопка оплаты картой/СБП не показывается.

## Реферальная скидка

Скидка применяется к цене конкретного периода.

Пример:

```text
30 дней: 100 ₽ - скидка
365 дней: 1000 ₽ - скидка
```

Скидка не должна ломать выбор периода.

## Лимит устройств/IP

Тариф задает лимит устройств/IP.

Этот лимит используется:

- в `🔑 Мои подписки`;
- в antiabuse;
- в warning;
- в auto-disable, если он включен.

Термин в UX:

```text
Лимит устройств/IP
```

## MultiHop / Double VPN

Это опциональный режим multihop-маршрутизации.

Возможные схемы:

```text
user → main server → remote server → internet
user → main server → remote server → WARP → internet
user → main server → WARP endpoint → internet
```

В тарифе можно включить или выключить этот режим.

Если режим выключен, в пользовательских экранах не нужно писать “недоступен”. Лучше просто не показывать строку.

## Трафик MultiHop / Double VPN

Для 30-дневных тарифов трафик может добавляться при продлении.

Для длинных тарифов может использоваться monthly reset:

```text
Лимит трафика Double VPN: 30 ГБ / Ежемесячно
```

В пользовательском UX не нужно писать технические “resetDays”, “save” и похожие слова.

## S-UI inbounds

Если MultiHop / Double VPN включен, тариф использует S-UI inbounds.

Если inbounds в тарифе не заданы, используется `sui.default_inbounds` из `config.json`.

Если режим выключен, inbounds не должны попадать в подписку.

## Доступные remote nodes

В тарифе можно выбрать:

- все node;
- только конкретные node;
- ни одной node.

Это влияет на блок “Локации в подписке” и на remote links.

Если локации не выбраны, блок “Локации в подписке” лучше скрывать.

## Протоколы node

Тариф выбирает node, а конкретные протоколы включаются на самой node:

```text
VLESS
HY2
TUIC
```

Если на node выключен TUIC, тариф не сможет получить TUIC link с этой node.

## Что проверить после настройки тарифа

```text
1. Тариф виден или скрыт в покупке как задумано.
2. Предпросмотр показывает правильное название.
3. Кнопка покупки использует отдельное название, если задано.
4. 30 дней отображается.
5. 365 дней отображается только если цена задана.
6. RUB-кнопки показываются правильно.
7. Stars-кнопки показываются правильно.
8. Реферальная скидка применяется.
9. Лимит устройств/IP отображается.
10. Remote локации отображаются.
11. MultiHop / Double VPN скрыт, если выключен.
12. Продление работает по тем же условиям.
```
