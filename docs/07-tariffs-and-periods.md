# Тарифы и периоды

Тариф определяет название, срок, стоимость, лимит устройств/IP, трафик, Double VPN, S-UI inbounds и remote node scope.

## Названия тарифа

Могут использоваться два названия: название для пользователя и название для страницы тарифов. Второе используется только как текст кнопки в покупке.

## Периоды

Тариф может иметь несколько сроков: 30 дней, 365 дней или пользовательский срок. Если цена для срока не задана, такой срок не показывается пользователю.

## Цены

Поддерживаются карта/СБП через внешний провайдер и Telegram Stars. Если Stars отключены или цена равна 0, Stars-кнопки не показываются.

## Referral discount

Реферальная скидка применяется к цене выбранного периода отдельно для каждого срока и способа оплаты.

## Лимит устройств/IP

Лимит используется в antiabuse и пользовательском UX. Везде рекомендуется термин “Лимит устройств/IP”.

## Double VPN

Если Double VPN выключен, пользователю лучше не показывать строку “недоступен”. Если включен, бот показывает трафик и условия сброса.

## S-UI inbounds

Тариф может задавать свои S-UI inbound IDs. Если они не заданы, используется `sui.default_inbounds` из `config.json`. Если Double VPN выключен, inbounds не используются.

## Node scope

Тариф может быть доступен на всех node, только на выбранных node или без remote node. Это влияет на remote links в подписке.
