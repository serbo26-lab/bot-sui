# Telegram proxy

Telegram proxy — отдельная функция для доступа к Telegram.

Он не заменяет VPN-подписку и не является VPN-протоколом. Это прокси для самого Telegram.

## На чем основан

В проекте используется mtg proxy.

Обычно бот устанавливает mtg на remote node из выбранного релиза и генерирует secret для подключения.

Рекомендуется фиксировать версию, например:

```text
v2.2.8
```

а не использовать latest без контроля.

## Где находится

Пользователь:

```text
💎 Telegram proxy
```

Админ:

```text
🛠 Админ → 💎 Telegram proxy
```

На node:

```text
🌍 Локации → node → 💎 Управлять TG PROXY
```

## Что делает пользовательская кнопка

Кнопка `💎 Telegram proxy` показывает готовые proxy-ссылки.

Пользователь нажимает ссылку и добавляет proxy в Telegram.

Если proxy недоступен пользователю, бот покажет сообщение с причиной.

## Кто может видеть proxy

Админ выбирает режим:

```text
только обычные подписки
обычные подписки + активный тестовый доступ
```

Если выбран первый режим, trial-пользователи proxy не видят.

Если выбран второй режим, активный trial тоже получает proxy.

## Настройка доступа

В админке можно показать/скрыть кнопку proxy, выбрать режим доступа, добавить proxy-запись, включить/выключить proxy, удалить proxy и изменить название.

## Установка на node

Типовой сценарий:

1. Админ открывает node.
2. Нажимает `💎 Управлять TG PROXY`.
3. Нажимает install.
4. Бот подключается к node по SSH.
5. Бот скачивает/устанавливает mtg.
6. Бот генерирует secret.
7. Бот выбирает domain/SNI.
8. Бот открывает firewall port, если включено.
9. Бот создает systemd service.
10. Бот добавляет proxy-запись в базу.
11. Пользовательская кнопка начинает показывать proxy.

## Domain/SNI

По умолчанию лучше использовать `node.domain`, например:

```text
node1.example.com
```

Если у node нет domain, админ должен указать domain/SNI вручную.

IP не должен быть основным/default SNI.

## Secret

Secret генерируется при установке proxy.

Если secret нужно заменить, админ может перегенерировать его. После этого пользователям нужно получить новую proxy-ссылку.

## Повторная установка

Если proxy уже установлен, бот не должен молча перезаписывать его.

Вместо этого должен появиться экран управления:

```text
Проверить
Переустановить
Перегенерировать secret
Удалить TG PROXY
```

## Managed records

Если proxy был создан ботом, повторная установка должна обновлять существующую managed-запись, а не создавать дубликаты.

## Названия proxy

Для пользователя лучше показывать понятные названия:

```text
🇳🇱 Telegram proxy NL
🇵🇱 Telegram proxy PL
🇩🇪 Telegram proxy DE
```

## Удаление

Удаление должно остановить service на node, удалить или отключить конфиг mtg, убрать managed record из базы и скрыть proxy из пользовательской кнопки.

## Что проверить

```text
1. Proxy install на node.
2. Повторный install показывает management screen.
3. Secret генерируется.
4. Proxy появляется в кнопке пользователя.
5. Режим paid_only скрывает proxy от trial.
6. Режим paid_or_trial показывает proxy active trial.
7. Удаление proxy убирает ссылку.
```
