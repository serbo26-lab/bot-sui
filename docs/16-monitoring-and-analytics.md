# Мониторинг и аналитика

Мониторинг показывает администратору, как используется бот и где есть проблемы.

## Где находится

```text
🛠 Админ → 📈 Мониторинг
```

## Периоды

Обычно используются:

```text
24ч
7д
30д
```

## Сводка

Сводка показывает краткую информацию: активные пользователи, новые пользователи, покупки, предпросмотры, клики оплаты, подписки, тесты, support, antiabuse, remote nodes и ошибки.

## Пользователи

Метрики:

```text
всего запускали бота
новые пользователи 24ч/7д/30д
активные пользователи 24ч/7д/30д
/start
купить
предпросмотр тарифа
оплата картой/СБП
оплата Stars
продление
тестовый доступ
FAQ
подключить устройство
профиль/рефералы
поддержка
недоступны для сообщений
```

## Админы не учитываются

Действия админов из `admin_ids` не должны попадать в пользовательскую статистику.

События могут храниться в базе, но при построении отчета фильтруются.

## Воронка покупки

Воронка помогает понять, где пользователь теряется:

```text
/start
↓
открыл купить
↓
открыл предпросмотр
↓
нажал оплату
↓
оплатил
↓
получил подписку
```

## Недоступны для сообщений

Telegram не сообщает напрямую, что пользователь удалил чат или заблокировал бота.

Метрика считается по ошибкам отправки:

```text
bot was blocked
chat not found
user is deactivated
forbidden
```

## Оплаты

Показывает pending, success, failed, apply_failed, provider statuses и суммы.

## Подписки

Может показывать активные, истекающие, продленные, отключенные, trial и manual/offline подписки.

## Remote nodes

Может показывать enabled nodes, sync errors, reconciliation status, node read errors и link issues.

## Antiabuse

Может показывать кейсы, предупреждения, ограничения, stale node data и ошибки сборщиков.

## Поддержка

Может показывать открытые/закрытые тикеты и активность поддержки.

## Почему нет backfill

Если мониторинг добавлен в новой версии, старые действия пользователей до установки не всегда можно восстановить.

Статистика обычно начинает наполняться с момента установки функции.

## Что проверить

```text
1. Админские действия не попадают в статистику.
2. /start считается.
3. Покупка считается.
4. Предпросмотр считается.
5. Клики оплаты считаются.
6. Trial считается.
7. Support считается.
8. Заблокировавший бот пользователь попадает в unreachable.
```
